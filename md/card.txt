
## Card

What is a Card(Contact Card)?
Card = Public Profile + Digital Identity + Communication + Trust + Payment Methods

* Public Profile
  Public-facing profile and personal information.

* Digital Identity
  Verified identity and account representation.

* Communication
  Message, Email, Meet, Website and contact channels.

* Trust
  Verification, Completed Collaborations.

  The Contact trust system consists of three layers：
  1. Identity verified
  2. Completed Deals and Rating / Review — Measures the quality of an Offer Template.
     (Custom Offer — No public reputation is created.)
  3. Honor — Represents trust in the person.

* Payment Methods
  Supported payment methods for receiving payments.

- Dash > Card is the management page for the user's public Contact Card (View + Share + Edit).
  - Display the user's public Card URL (card.contact/{username}).
  - Provide a Public Preview that opens the user's public Contact Card.
- Provide quick actions:
  - Edit Card
  - Share Card
  - Copy Link
  - QR Code
- Native Share opens the operating system share sheet when supported.
- QR Code always represents the current public Card URL.
- Download PNG exports the current QR Code as a PNG image.
- Changes are reflected immediately on the public Card.


## Public Card Philosophy

Contact Card is intended to replace traditional personal and company websites.
The Card should become the user's primary public identity and communication page.
External links should only supplement information not already provided by Contact.

## Add Link

Users should never be required to manually select the platform before entering a URL.
The system automatically detects the platform after the URL is entered.
Only unknown platforms require a custom title.


## Offer Activation Requirements

Offers are disabled by default.
Users must complete account verification before Offers can be enabled.

Activation rules:

Personal Account
- Account Type = Personal
- Identity Verification is required.

Organization Account
- Account Type = Organization
- Organization Verification is required.

If verification is incomplete:
- The Offers section cannot be enabled.
- Users attempting to add their first Offer should be redirected to Verification.

After the required verification is completed:
- The Offers section becomes available automatically.


## First Offer Flow

When the user selects:

+ Add Offer

for the first time, the system checks whether the required verification has been completed.

If verification has not been completed:
Personal Account
Redirect to:
Verification > Identity
Organization Account
Redirect to:
Verification > Organization

The Offer editor must not open until the required verification has been approved.


## Offers Availability

Offers remain hidden until the required verification has been completed.

Before activation:

After activation:

## Verification

Verification
├── Identity Verification
├── Organization Verification
├── Expertise Verification
└── Verification Synchronization

Verification helps build trust with customers and partners.
Verification consists of three independent verification types:
- Identity
- Organization
- Expertise

Each verification has its own lifecycle.
Each identity must be unique and cannot be verified by multiple accounts.

Verification states:
- Not Verified
- In Review
- Verified
- Rejected

Only Verified status displays a public verification badge.
Users can retry verification after rejection.

Verification status is synchronized across:
- Dashboard
- Public Contact Card
- Offer
- Deal


### Identity Verification

Identity verification confirms that the account belongs to a real individual.

Required verification steps:
- Verified email
- Verified phone number
- Government-issued identity document
- Face verification

Identity verification is completed only after all required information has been reviewed and approved.

After approval:
✓ Identity Verified

---

### Organization Verification

Organization verification confirms that the organization represented by the account is legitimate.

Required verification steps:
- Organization legal name
- Business registration document
- Tax identification number
- Official website (optional)

Organization verification is completed only after review and approval.

After approval:
✓ Organization Verified

---

### Expertise Verification

Expertise Verification is optional.
It is intended to increase professional credibility and trust.
Users are not required to complete Expertise Verification in order to use Offers.
Expertise Verification confirms professional capability through manual review by Contact.

Verification process:
- Select expertise category.
- Submit portfolio.
- Submit professional experience.
- Submit supporting credentials (optional).
- Schedule a live interview with Contact.
- Complete manual review by Contact.

Possible verification states:
- Request Verification
- Verification In Review
- Verified
- Rejected

Expertise Verification is completed only after both the interview and manual review have been approved.

After approval:

Display:
✓ Expertise Verified
The verified professional category may also be displayed.

Example:
✓ Expertise Verified
Patent Attorney


## Verification Synchronization

Verification status is a global account property.
Updating verification automatically updates every place where verification is displayed, including:
- Dashboard
- Public Contact Card
- Messages
- Live Mail
- Live Chat
- Offers
- Deals
- Future public profile surfaces

Verification information must never be duplicated across products.

## Public Contact Card

* Public Card URL
- Generate public Card URL based on user's username.
- Format: card.contact/{username}
- Public Card is the main identity page for each user.

* Card Privacy
- Users can control visibility of profile sections.
- Hidden sections should not appear on public Card.

* Card Sharing
- Display Copy Link and QR Code actions in the Card header.
- Copy Link copies the public Card URL to the clipboard.
- QR Code encodes the public Card URL: card.contact/{username}.
- Both actions always reference the current public Card URL.

* Availability
- Display user's availability status next to the user's name.
- Availability is controlled by user's account settings.
- Status options:
  - ON
  - Away
  - Busy
  - DND
  - OFF
- Available means the user is accepting new messages and meeting requests.
- This status does not represent real-time online presence.

* Response Time
- Calculate average response time based on user's message history.
- Display automatically.
- Do not allow manual editing.

* Identity & Business Verification
- Display verification badges based on user's verification status.
- Show "✓ Identity verified" when identity verification is completed.
- Show "✓ Business verified" when business verification is completed.
- Both badges can be displayed at the same time.

* Completed Collaborations
- Completed Collaborations = Number of completed Orders confirmed by both parties.
- Trigger: When order.status changes to COMPLETED and both parties confirm completion:
  completed_collaborations += 1
- Each completed Order can only increment Completed Collaborations once.

* Message
- Generate public message URL based on user's username.
- Format: talk.contact/{username}
- Visitors can start a conversation without login.
- New conversations are added to the user's Inbox Timeline.

* Message Requests
- Messages from unknown users should be treated as new conversation requests.
- Users can accept, block, or ignore requests.

* Meeting
- Generate public meeting URL based on user's username.
- Format: meet.contact/{username}
- Visitors can view available meeting slots and request/book meetings.

* Email
- Generate default Contact email based on username.
- Format: {username}@livemail.me
- Support additional user email addresses.
- Display primary email first.
- Allow users to control email visibility.

* Payment Methods
- Display available payment methods inline.
- Limit display to 2 rows.
- Show "+{remaining_count} more" when exceeded.
- Clicking a payment method opens payment details.
- Payment methods are loaded from the user's connected payment providers.
- Payment methods are only displayed when enabled by the user.

* Links
- Users can add multiple external links.
- Contact Card itself serves as the user's primary public website.
- External links are intended only for resources not provided by Contact.
- Supported examples include:
  - Website
  - GitHub
  - LinkedIn
  - Behance
  - Dribbble
  - YouTube
  - App Store
  - Google Play
  - Portfolio
  - Documentation
  - Other external resources

* Add Link
- Users only enter a URL.
- The system automatically detects:
  - Platform
  - Title
  - Icon
  - Domain

Example:
Input:
https://github.com/john
Automatically becomes:
GitHub
github.com/john

- If the platform cannot be recognized:
  - Ask the user to enter a custom title.

* Link Display
- Display the primary link first.
- Limit display to 2 rows.
- Show "+{remaining_count} more" when exceeded.
- Hide the section when no links are available.

* Contact Information
- Support multiple email addresses and external links per user.
- Display available information based on user's privacy settings.
- Hide sections when no items are available.

* Verification Badges
- Verification badges are clickable.
- Clicking any verification badge opens the user's public verification page.
- Public verification page displays all completed verification types.


## Honor

Honor is a public trust signal awarded by verified contacts.

Unlike ratings, Honors recognize long-term trust between people rather than the quality of a specific Deal.

Honors belong to the person, not to an Offer Template or Deal.

### Award Honor Entry Points

Users can access the Award Honor flow from:

- Contact menu
- Public Contact Card
- Completed Deal review flow

### Award Rules

- Only Verified Identity users can award Honors.
- One Honor can be awarded every 60 days.
- The same contact can receive another Honor after 90 days.
- Honors can be revoked individually.
- Revoking one Honor adds an additional 60 days to the user's current Honor cooldown.
- Honor cooldown extensions are cumulative.

### Honor Display

Public Contact Cards display:

- Total Honors Received
- Number of unique verified contacts who awarded Honors
- Honor history

Example:
🏅 27 Honors Received
From 20 Verified Contacts

### Purpose

Honors provide long-term public trust.

Unlike ratings:
- Honors belong to people.
- Ratings belong to Offer Templates.
- Completed Collaborations represent business experience.

These three trust signals are intentionally independent.
